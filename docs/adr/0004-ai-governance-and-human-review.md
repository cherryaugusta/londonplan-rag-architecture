# ADR 0004: AI governance and human review

## Status
Accepted

## Context
The long-term platform direction includes RAG and planning-change summaries. These outputs require traceability and human review.

## Decision
We will version prompt catalogs and sample evaluation records in the repository and require human review for planning summaries and citation-bearing answers.

## Consequences
- Governance becomes visible in the codebase
- Review workflows are easier to explain to stakeholders
- The project shows AI-adjacent responsibility, not only model usage
